ReservaMate — Mt. ClaRamuel Resort Reservation System
Business Problem

Mt. ClaRamuel Resort was running on paper notebooks and phone calls. Staff logged bookings in spreadsheets, collected payments in cash or bank transfer, and reconciled everything manually at day's end. This produced chronic double-bookings, lost payment records, and a full-time admin burden — while competitors with digital presence were capturing online customers the resort couldn't reach.

Solution Delivered

ReservaMate is a production-grade reservation platform co-built with Edcel B. Yasay for my BS IT capstone (ISU Ilagan, December 2025). Five core modules replaced the manual workflow entirely.

  • ▸Real-time booking engine — concurrent-safe scheduling that prevents double-bookings as guests transact simultaneously.
  • ▸Multi-payment gateway — Maya Business integration covering e-wallets, credit and debit cards with automated receipts and transaction records linked directly to bookings.
  • ▸Admin dashboard with RBAC — role-gated access for staff tiers, full booking oversight, transaction monitoring, and user management.
  • ▸Check-in / check-out module — real-time occupancy updates so facility turnaround stays accurate for incoming guests.
  • ▸Financial reporting — filterable revenue reports by month or year across room types, replacing the manual ledger entirely.

Key Technical Decisions

Laravel's ORM and authentication scaffolding accelerated development while keeping the codebase maintainable. MySQL with concurrency-safe writes handles simultaneous bookings without race conditions. Maya Business was chosen specifically because it supports Philippine e-wallets natively — the dominant payment method for the resort's local customer base. Security was layered: RBAC at the application layer, password hashing, SSL, and encrypted data storage.

CIMC Patient Record Viewer
Business Problem

Hospital staff at CIMC retrieved patient records from physical folders stored across multiple rooms. A single patient's history could take 15–30 minutes to locate — an unacceptable delay in urgent care situations and a full-time burden on a dedicated records clerk.

Solution Delivered

A digital archiving and search system that digitized decades of physical records into a secure SQL Server database. Staff can now search by patient name, ID, or admission date and retrieve a complete record in under 30 seconds. The system includes PDF document rendering, annotation, and role-based access control to restrict sensitive data to authorized staff only.

Key Technical Decisions

SQL Server was chosen to align with the hospital's existing IT infrastructure. Full-text indexing on name and patient ID fields keeps search near-instant even as the dataset grows. PDF rendering is handled server-side to avoid exposing raw file paths to the client — an important security constraint for regulated medical data.

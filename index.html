Case Study — ReservaMate: Web-Based Reservation System for Mt. ClaRamuel Resort
Context

ReservaMate is my BS Information Technology capstone project, co-developed with Edcel B. Yasay at Isabela State University – Ilagan Campus (December 2025). It was built to production spec, deployed live at Mt. ClaRamuel Resort and Events Place, and formally evaluated by 51 respondents against the ISO/IEC 25010 Software Quality Model.

The Problem

Mt. ClaRamuel Resort operated entirely on manual workflows: customers called, texted, or walked in to check availability; staff logged bookings in notebooks or spreadsheets; payments were collected in cash or bank transfer and stored separately from booking records. This produced four documented failure modes:

  • ✕Inconsistent data management — discrepancies between booking logs, payment records, and customer info regularly caused overbooking and scheduling conflicts.
  • ✕Manual payment tracking — no integrated system across cash, bank transfers, and mobile wallets led to delayed confirmations, lost receipts, and difficult financial reconciliation.
  • ✕Limited digital visibility — no online booking platform meant the resort missed customers who search and book digitally, falling behind competitors with digital infrastructure.
  • ✕Time-consuming report generation — staff manually compiled data from multiple sources, causing delays in financial decision-making and increasing accounting errors.
What I Built — Five Core Modules
  • ▸Customer Booking Interface — real-time facility browsing with a scheduling tool per facility; guests select dates and guest types for accurate pricing. Reservations sync instantly to the DB to prevent concurrent double-bookings.
  • ▸Multi-Payment Gateway — Maya Business integration supporting, e-wallet, and credit/debit cards. Automated real-time payment verification, instant digital receipts, and transaction records linked directly to bookings.
  • ▸Admin Dashboard — full CRUD for user and admin accounts, role-based access control (RBAC), booking oversight, and transaction monitoring from a central interface.
  • ▸Staff Check-in / Check-out Module — real-time occupancy status updates so facility turnaround and room inventory are always accurate for incoming guests.
  • ▸Financial Reporting Module — filterable revenue reports by month or year covering room reservations and facility accommodations, supporting data-driven pricing and seasonal planning decisions.

Key Technical Decisions

Laravel (PHP) was chosen for the backend due to its robust ORM, built-in authentication scaffolding, and mature ecosystem for rapid API development. MySQL handles relational booking data with query optimization for concurrent access. Maya Business was selected as the payment gateway because it natively supports Philippine e-wallets (Maya) — the dominant payment methods for the resort's local customer base. Security was implemented through password hashing, encrypted data storage, SSL encryption, and RBAC at the application layer. The iterative waterfall methodology structured development across six phases: requirements gathering, system design, security implementation, testing, deployment, and maintenance.

Case Study — CIMC Patient Record Viewer
The Problem

Hospital staff at CIMC were retrieving patient records from physical folders stored across multiple rooms. Finding a single patient's history could take 15–30 minutes, causing delays in urgent care situations and requiring a dedicated records clerk full-time.

What I Built

A digital archiving and search system that ingested decades of physical records into a secure SQL Server database. Features include full-text patient search by name, ID, or admission date, PDF document rendering and annotation, and role-based access to restrict sensitive data to authorised staff only.

Key Technical Decisions

Used SQL Server to align with the hospital's existing IT infrastructure. Implemented full-text indexing on patient name and ID fields to keep search near-instant even at scale. PDF manipulation was handled server-side to avoid exposing raw file paths to the client.
